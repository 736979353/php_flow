## 2. 插件简介

交易流程协同管理插件是一套基于 FastAdmin 开发的后台业务流程插件，主要用于交易类、合同类、代书类、资料流转类等需要多人协同办理的业务场景。

插件提供可视化流程设计、表单模板设计、审核节点、表单节点、节点抄送、并行办理、流程发起、流程办理、流程详情查看、附件预览、权限分配和外部业务调用能力。

通过该插件，后台管理员可以不修改代码，直接在后台设计交易流程和节点表单。业务人员可以按照流程逐步或并行办理节点，相关抄送人员可以实时查看流程进度和每一步填写的信息。

## 3. 应用场景

### 3.1 交易订单审批

适用于交易订单从创建、资料填写、主管审核、财务确认、风控复核到最终完成的完整流程。

### 3.2 合同和代书业务流转

适用于合同资料、客户资料、交易资料、附件材料在多个岗位之间流转办理。

### 3.3 多部门同步办理

适用于财务、风控、业务、运营等多个部门需要同时处理同一笔业务的场景。

### 3.4 表单资料收集

适用于提前创建标准表单模板，例如客户信息表、交易信息表、费用明细表、资料清单表等。

### 3.5 流程进度跟踪

适用于发起人、审核人、抄送人、管理人员需要随时查看流程进展和节点内容的场景。

### 3.6 其他业务模块发起流程

适用于订单、客户、合同、项目等其他业务模块在创建数据后，自动发起一个交易流程。

## 4. 解决的问题

| 问题 | 插件解决方式 |
| --- | --- |
| 流程靠人工沟通推进 | 使用后台流程节点驱动，每个节点有明确办理人和状态 |
| 表单格式不统一 | 通过表单模板统一字段和填写规则 |
| 审批步骤固定死板 | 支持串行流程，也支持多节点并行办理 |
| 过程不可追踪 | 记录节点填写内容、操作人、操作时间和办理日志 |
| 抄送人无法了解进度 | 抄送人可查看流程进度和每一步表单信息 |
| 附件资料分散 | 节点内上传附件，详情页集中查看和预览 |
| 权限边界不清晰 | 按发起人、办理人、抄送人、被分配人、全量查看组控制可见范围 |
| 其他模块重复开发审批 | 提供服务类调用能力，其他控制器可以直接发起流程 |

## 5. 功能列表

### 5.1 流程设计

- 支持在后台创建流程设计。
- 支持可视化拖拽节点。
- 支持开始节点、结束节点、审核节点、表单节点。
- 支持节点位置保存。
- 支持节点之间自动连线展示。
- 支持同一阶段多个节点同步办理。
- 支持流程启用、停用、草稿状态。

### 5.2 审核节点

- 审核节点可以指定审核人。
- 审核人可以在流程办理页面处理对应节点。
- 节点办理记录会写入日志。
- 审核节点可以配置抄送人。

### 5.3 表单节点

- 表单节点可以选择表单模板。
- 选择模板后自动应用模板字段。
- 节点内不需要重复手动添加模板字段。
- 发起和办理流程时，表单节点按字段配置生成填写界面。

### 5.4 表单模板

- 支持独立的表单模板菜单。
- 支持创建、编辑、删除表单模板。
- 支持可视化表单设计。
- 支持字段拖拽、排序、配置。
- 支持模板复用到不同流程节点。

### 5.5 表单字段类型

| 字段类型 | 说明 |
| --- | --- |
| 信息填写 | 适合姓名、编号、金额、标题等单行内容 |
| 多行文本 | 适合备注、说明、审核意见等长文本 |
| 下拉选择 | 适合从固定选项中选择一个值 |
| 单选 | 适合多个选项中选择一个 |
| 多选 | 适合多个选项中选择多个 |
| 附件上传 | 适合上传图片、PDF、Word、Excel 等资料 |
| 时间 | 适合选择日期或时间 |
| 数组明细 | 适合交易明细、费用明细、资料清单等多行结构化数据 |

### 5.6 数组明细字段

- 数组字段支持多个子字段。
- 子字段包含标题、类型、内容。
- 子字段支持文本、下拉、单选、多选、附件、时间等类型。
- 数组字段可以录入多行数据。
- 数组字段内可以上传附件。
- 详情页以易读方式展示，不直接显示原始 JSON。

### 5.7 流程发起

- 可在流程设计列表中直接点击“发起流程”。
- 一个流程设计默认只能发起一个流程办理。
- 已发起流程可以在流程设计列表展开查看。
- 展开区域支持查看、办理、删除流程办理。
- 删除流程办理后可以重新发起。
- 发起流程时可以选择额外抄送人。
- 支持通过 URL 参数传入抄送人，例如 `cc_ids=3,6`。

### 5.8 流程办理

- 办理人可以查看自己需要处理的节点。
- 并行阶段的多个节点可以同时办理。
- 节点提交后仍然可以显示。
- 已填写节点可以继续补充或修改。
- 修改后保存最新内容。
- 每次办理和修改可记录操作人。

### 5.9 流程详情

- 详情页左侧可视化显示所有节点。
- 点击节点可以查看该节点表单信息。
- 可以区分已填写字段和未填写字段。
- 表单内容以易读方式展示。
- 数组字段按结构化内容展示。
- 附件支持图片、PDF、Word、Excel 等查看或预览。
- 显示流程办理日志和节点进度。

### 5.10 抄送机制

- 每个节点可配置抄送人。
- 发起流程时可传入额外抄送人。
- 抄送人可以查看流程进度。
- 抄送人可以查看节点表单信息。
- 抄送人默认不参与节点办理。

### 5.11 权限控制

- 普通用户只能查看自己相关的流程。
- 办理人可以查看自己需要办理的流程。
- 抄送人可以查看被抄送的流程。
- 被分配用户可以查看分配给自己的流程设计。
- 插件配置中可选择“全量查看用户组”。
- 全量查看权限只对直接属于该用户组的管理员生效。
- 全量查看权限不会自动继承给下级用户组。

### 5.12 批量分配

- 流程设计列表支持勾选多条流程。
- 支持批量分配给指定下级用户。
- 被分配用户可以看到对应流程设计。
- 未被分配用户不能越权查看。

### 5.13 外部业务调用

其他控制器可以通过服务类发起流程。

示例：

```php
use app\admin\library\tradeflow\Service as TradeflowService;

$instance = TradeflowService::launch(2, [
    'title'        => '交易流程标题',
    'initiator_id' => $this->auth->id,
    'form_data'    => [
        'field_name' => '测试内容',
    ],
    'cc_ids'       => [3, 6],
]);
```

说明：

- 不建议手动实例化后台控制器。
- 推荐通过 `app\admin\library\tradeflow\Service` 调用。
- 可传入流程设计 ID、标题、发起人 ID、初始表单数据、额外抄送人。

## 12. 配置说明

插件配置项主要包括：

| 配置项 | 说明 |
| --- | --- |
| 全量查看用户组 | 选择后，该用户组内管理员可以查看所有流程设计和流程办理 |

## 13. 使用流程

### 13.1 创建表单模板

1. 进入表单模板菜单。
2. 点击新增。
3. 填写模板名称和说明。
4. 使用可视化表单设计器添加字段。
5. 配置字段标题、类型、选项、数组子字段等。
6. 保存模板。

### 13.2 创建设计流程

1. 进入流程设计菜单。
2. 点击新增流程。
3. 填写流程名称和说明。
4. 进入可视化设计页面。
5. 添加审核节点和表单节点。
6. 配置审核人、抄送人和表单模板。
7. 拖拽节点调整办理顺序和并行关系。
8. 保存并启用流程。

### 13.3 发起流程

1. 进入流程设计列表。
2. 在操作列点击“发起流程”。
3. 填写流程标题。
4. 选择额外抄送人。
5. 提交发起。

### 13.4 办理流程

1. 办理人进入流程办理页面。
2. 打开待办流程。
3. 点击对应节点。
4. 填写表单内容或审核意见。
5. 上传附件。
6. 提交节点。

### 13.5 查看详情

1. 进入流程详情页面。
2. 左侧查看全部节点。
3. 点击节点查看字段内容。
4. 查看已填和未填信息。
5. 查看附件和办理日志。

## 14. 权限说明

插件的数据可见范围主要由以下身份决定：

| 身份 | 可见范围 |
| --- | --- |
| 发起人 | 可以查看自己发起的流程 |
| 办理人 | 可以查看和办理分配给自己的节点 |
| 抄送人 | 可以查看被抄送流程的进度和表单信息 |
| 被分配用户 | 可以查看分配给自己的流程设计 |
| 全量查看组用户 | 可以查看全部流程设计和流程办理 |

权限设计原则：

- 普通用户不能查看无关流程。
- 抄送人只查看，不默认参与办理。
- 全量查看组不向下级自动继承。
- 流程设计可以批量分配给下级用户。

| 附件类型 | 支持情况 |
| --- | --- |

| PDF | 支持点击预览 |

| Excel | 支持预览或打开查看，取决于系统预览能力 |

## 16. 外部集成说明

插件提供服务类，便于项目其他控制器发起流程。

服务类路径：

```php
app\admin\library\tradeflow\Service
```

调用示例：

```php
use app\admin\library\tradeflow\Service as TradeflowService;

$instance = TradeflowService::launch(2, [
    'title'        => '交易流程标题',
    'initiator_id' => $this->auth->id,
    'form_data'    => [
        'customer_name' => '张三',
        'amount'        => '100000',
    ],
    'cc_ids'       => [3, 6],
]);
```

参数说明：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `flowId` | int | 是 | 流程设计 ID |
| `title` | string | 是 | 流程实例标题 |
| `initiator_id` | int | 是 | 发起人管理员 ID |
| `form_data` | array | 否 | 初始表单数据 |
| `cc_ids` | array/string | 否 | 额外抄送人 ID |

返回值：

- 返回创建成功的流程实例对象。
- 如果流程不存在、未启用、已发起过或参数错误，会抛出异常。

## 17. 特点和优势

### 17.1 更贴近交易业务

插件不是单纯的审批流，而是围绕交易资料、审核节点、表单填写、附件上传、抄送查看和并行办理设计，更适合实际交易业务。

### 17.2 可视化配置，减少代码修改

管理员可以在后台配置流程和表单模板，减少每次新增流程都需要开发人员改代码的问题。

### 17.3 支持并行办理

多个节点可以同步处理，适合财务、风控、业务等多个角色同时参与的场景。

### 17.4 表单模板可复用

常用表单可以沉淀为模板，不同流程节点可以直接复用，降低重复配置成本。

### 17.5 过程留痕

流程办理、节点提交、内容修改都可以记录，方便后续追踪责任和查看历史。

### 17.6 抄送透明

相关人员即使不参与办理，也可以查看流程进度和节点资料，减少重复沟通。

### 17.7 权限边界明确

通过普通可见范围、抄送范围、分配范围、全量查看用户组控制数据权限，减少越权查看问题。

### 17.8 易于集成

其他业务模块可以通过服务类发起流程，适合和订单、客户、合同、项目等模块联动。

### 17.9 沿用 FastAdmin 体系

菜单、权限、表格、表单、弹窗、上传等能力沿用 FastAdmin 原有体系，后台用户学习成本低，二次开发也更方便。

## 18. 注意事项

- 修改插件后需要确认运行目录和 `addons/tradeflow` 插件目录同步。
- 浏览器页面无变化时，建议执行 `Ctrl+F5` 强制刷新。
- 已安装过旧版本的项目，如果新增字段未生效，需要检查数据库表结构是否已更新。
- 外部业务调用时不要手动实例化控制器，推荐使用服务类。
- 全量查看用户组配置后，需要确认管理员和用户组的直接绑定关系。
- Word、Excel 预览能力依赖实际部署环境的文件预览能力。

## 19. 常见问题

### 19.1 为什么流程设计选择表单模板后看不到字段？

这是当前设计逻辑。表单节点选择模板后会自动应用模板字段，节点里不再展示字段明细，也不允许在节点里删除模板字段，避免流程节点和模板字段不一致。

### 19.2 为什么一个流程设计只能发起一个流程办理？

当前业务规则要求一个流程设计只能存在一个办理流程。如果需要重新发起，可以在流程设计列表展开已发起流程，先删除旧办理，再重新发起。

### 19.3 抄送人能不能修改节点内容？

默认不能。抄送人的定位是查看进度和表单信息，不参与节点办理。

### 19.4 下级用户为什么看不到上级的全部流程？

全量查看权限不会自动继承给下级。如果下级需要查看指定流程设计，需要通过批量分配功能分配给该下级用户。

### 19.5 其他模块如何发起流程？

使用 `app\admin\library\tradeflow\Service` 的 `launch()` 方法发起，不建议直接调用或实例化控制器。
